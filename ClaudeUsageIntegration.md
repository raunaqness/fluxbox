# Claude Usage Integration Guide

This document outlines how to integrate the Claude token usage tracking feature into the `mac-menu-bar-app` (Tauri).

## Overview
The feature works by piggybacking on the OAuth token generated by the **Claude Code** CLI tool. This allows for a "zero-config" setup where the app automatically detects the user's session.

---

## 1. Authentication (macOS Keychain)
The OAuth token is stored by Claude Code in the macOS Keychain.

- **Service Name:** `Claude Code-credentials`
- **Account Name:** `claudeai_oauth_token` (used as the primary key)
- **Data Format:** A JSON object containing:
  ```json
  {
    "claudeAiOauth": {
      "accessToken": "sk-ant-...",
      "expiresAt": 1744525634.123
    }
  }
  ```

### Implementation (Rust/Tauri)
Use the `keyring` crate to fetch this secret:
```rust
use keyring::Entry;

fn get_claude_token() -> Result<String, Box<dyn std::error::Error>> {
    let entry = Entry::new("Claude Code-credentials", "claudeai_oauth_token")?;
    let secret = entry.get_password()?;
    // Parse JSON and return accessToken
}
```

---

## 2. API Reference
The app calls an internal Anthropic endpoint.

- **URL:** `https://api.anthropic.com/api/oauth/usage`
- **Method:** `GET`
- **Required Headers:**
  - `Authorization: Bearer <ACCESS_TOKEN>`
  - `anthropic-beta: oauth-2025-04-20` (Critical: The API will fail without this specific version string)

### Response Schema
```json
{
  "five_hour": {
    "utilization": 45.5,
    "resets_at": "2026-04-13T18:30:00.000Z"
  },
  "seven_day": {
    "utilization": 72.1,
    "resets_at": "2026-04-16T12:00:00.000Z"
  },
  "seven_day_sonnet": {
    "utilization": 20.0,
    "resets_at": "2026-04-16T12:00:00.000Z"
  }
}
```

---

## 3. Implementation Workflow

### Step A: Backend (Rust)
1.  **Define Structs:** Create Serde-compatible structs for the JSON response and the Keychain data.
2.  **Create Command:** Implement a Tauri command `fetch_claude_usage`.
3.  **HTTP Client:** Use `reqwest` to perform the fetch with the specified headers.

### Step B: Frontend (TypeScript)
1.  **State Management:** Store the usage data in a global store (e.g., Zustand or a simple React context).
2.  **Polling:** 
    - Poll every **5 minutes** while the app is active.
    - If a **429 (Rate Limit)** error occurs, increase the interval to **15 minutes**.
3.  **Display:**
    - Show the 5h usage as a primary metric.
    - Show the weekly total in the detail view.
    - Calculate "Time Until Reset" by comparing `resets_at` with the current system time.

---

## 4. Nuances & Warnings
- **Dependency:** Users must have run `claude` and logged in at least once on their machine.
- **Undocumented API:** This endpoint is not part of the public Anthropic API and may be deprecated or changed by Anthropic without notice.
- **Rate Limits:** Anthropic is strict about usage polling. Do not poll more frequently than every 2-3 minutes.
